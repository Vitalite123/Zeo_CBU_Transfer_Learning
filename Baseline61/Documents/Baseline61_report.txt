2. Model Principles and Algorithms
2.1 XGBoost Model
2.1.1 Principle Overview
XGBoost is an efficient implementation based on Gradient Boosting Decision Trees (GBDT), which improves prediction performance by constructing an ensemble of multiple weak learners (decision trees).
2.1.2 Core Algorithm
The objective function of XGBoost consists of two parts:
plaintext
Obj(Θ) = L(Θ) + Ω(Θ)
Where:
L(Θ): Loss function that measures model fitting error
Ω(Θ): Regularization term that controls model complexity
For the t-th iteration, the objective function can be expressed as:
plaintext
Obj^(t) = Σi L(yi, ŷi^(t-1) + ft(xi)) + Ω(ft)
Using Taylor expansion approximation, we get:
plaintext
Obj^(t) ≈ Σi [L(yi, ŷi^(t-1)) + gi ft(xi) + 1/2 hi ft^2(xi)] + Ω(ft)
Where:
gi: First derivative (gradient) of the loss function
hi: Second derivative (Hessian) of the loss function

Regularization: L1 and L2 regularization prevent overfitting
Parallel processing: Supports multi-threaded parallel computing
Missing value handling: Automatically handles missing values
Pruning strategy: Uses maximum depth instead of number of leaf nodes
Approximate algorithm: Supports approximate splitting for sparse data
2.2 RandomForest Model
2.2.1 Principle Overview
RandomForest is an ensemble learning method based on Bagging (Bootstrap Aggregating), which makes predictions by constructing multiple decision trees and conducting voting.
2.2.2 Core Algorithm
RandomForest training process:
Bootstrap sampling: Randomly extract n samples with replacement from the training set to generate n training subsets
Random feature selection: Randomly select m features (m < M) when splitting each node
Decision tree construction: Build fully grown decision trees using CART algorithm
Ensemble prediction: For classification problems, adopt majority voting

2.3 LightGBM Model
2.3.1 Principle Overview
LightGBM is a gradient boosting framework developed by Microsoft, adopting Histogram-based decision tree algorithm and GOSS (Gradient-based One-Side Sampling) technology.
2.3.2 Core Algorithm
1. Histogram-based decision tree algorithm:
Discretize continuous feature values into k bins
Build histograms using bin indices
Select optimal split points based on histogram split gain
2. GOSS (Gradient-based One-Side Sampling):
Retain samples with large gradients (high information content)
Randomly sample samples with small gradients
Compensate sampling bias using constants
3. EFB (Exclusive Feature Bundling):
Identify mutually exclusive features (rarely non-zero simultaneously)
Bundle multiple mutually exclusive features into one feature

6.4 Evaluation Metric Explanation
Accuracy
plaintext
Accuracy = (TP + TN) / (TP + TN + FP + FN)
Measures the proportion of overall correct predictions.
F1 Weighted
plaintext
F1 Weighted = Σ (Weight_i × F1_i)
Considers class imbalance and calculates F1 score weighted by sample size.
F1 Macro
plaintext
F1 Macro = (1/N) × Σ F1_i
Treats all classes equally without considering sample size differences.
Variance (Prediction Variance)
plaintext
Variance = Var(Probabilities)
Measures the uncertainty of model predictions, lower variance indicates more confident predictions.

8.1 Complete Metric Definitions
This section supplements the missing Precision and Recall metrics in the previous report to provide a more comprehensive model evaluation.
8.1.1 Precision
plaintext
Precision = TP / (TP + FP)
Measures the proportion of positive predictions that are actually positive.
Precision Macro: Average of Precision across all classes, with equal weight for each class
Precision Weighted: Weighted average of Precision by sample size of each class
8.1.2 Recall
plaintext
Recall = TP / (TP + FN)
Measures the proportion of actual positive samples that are correctly predicted.
Recall Macro: Average of Recall across all classes, with equal weight for each class
Recall Weighted: Weighted average of Recall by sample size of each class
